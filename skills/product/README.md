# Product 板块

做产品判断时的方法论。**这是 Zac 独有的板块，Waza 没有。**

## 待建 Skills

| Skill | 用途 | 痛点 |
|---|---|---|
| research | 研究外部产品/项目 | AI 默认泛泛分析，不问"开发者为什么这么做"，不区分"记忆系统 vs 记录系统" |
| decide | 产品决策（要不要学 X 的做法） | AI 只会列选项，不表态 |
| service-design | 芦苇服务设计 | 七条硬约束 + 服务设计三条纪律 + TASTE |
| tagline | 产品叙事/标语 | AI 默认写"更好、更快、更强"的空话 |

## 设计理念

产品方法论的核心是 **"Zac 认为一个资深产品人应该怎么判断"**——不是"产品经理一般怎么做"。

每个 skill 应该包含：
- **硬约束**（必须做什么）
- **禁止项**（常见的错误做法）
- **Gotchas 表**（真实踩过的坑 + 对应规则）
- **自我欺骗检查表**（AI 典型的产品合理化思维）
- **输出格式**（不留模糊）
